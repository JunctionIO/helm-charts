# junction

Helm chart for the Junction webhook/event routing platform.

## Components deployed

| Component | Kind |
|---|---|
| API (nginx + PHP-FPM) | Deployment + Service |
| HTTP destination worker | Deployment |
| System/internal worker | Deployment |
| CLI migration | Job (post-install/pre-upgrade hook) |
| CLI token generation | Job (manual, disabled by default) |
| Postgres | StatefulSet + Service (official `postgres` image) |
| RabbitMQ | StatefulSet + Service (official `rabbitmq` image) |

## Required values

`secrets.app.jwtSecret` and `secrets.worker.token` have no default and must be set explicitly, e.g.:

```bash
helm install junction . \
  --set secrets.app.jwtSecret=$(openssl rand -hex 32) \
  --set secrets.worker.token=<token issued via the CLI keygen Job>
```

The worker token must be generated by running the CLI's `token:create` command against a database this release can already reach — see the `cli.keygen` section in `values.yaml` for the manual Job invocation.

## External Postgres/RabbitMQ

Set `postgres.enabled: false` / `rabbitmq.enabled: false` and populate `database.host` / `queue.host` (and adjust `secrets.app` credentials) to point at externally-managed instances instead of the bundled StatefulSets.

Postgres and RabbitMQ are bundled directly (official `postgres`/`rabbitmq` images, not a Bitnami subchart) — Bitnami's free-tier images either have no reproducible pinned tags left (Postgres) or no public tags at all (RabbitMQ), a result of Broadcom restricting older Bitnami image tags to a paid tier.

## Resource sizing

All `resources` blocks in `values.yaml` are best-guess placeholders for the POC — not yet benchmarked under load.
